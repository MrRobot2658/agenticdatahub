01

多渠道识别合并

02

热 + 冷双写架构

03

可观测 + 双灌入

merge_log 合并审计、画像实时更新。API 直连 + Kafka 异步两种灌入路径，灵活适配不同场景。

传统 CDP vs AgenticDataHub
08 / 10
COMPARISON · 06

从「数据管道」到「智能底座」

A 传统 CDP
数据管道

数据进来、存好、导出。人需要手动写 SQL 圈人、手动配旅程、手动看报表。

  • SQL 手写圈人，门槛高
  • 批量 T+1 画像更新
  • 触达规则手动编排
  • 分析看板需 BI 工具
  • 无 AI 能力，纯规则引擎
B AgenticDataHub
智能底座

数据进来、实时归一、AI Agent 直接操作。自然语言圈人、自动编排、一句话出报表。

  • 自然语言圈人 → DSL → SQL
  • 实时 Flink 画像更新 <3s
  • AI 旅程 Journeys 智能编排
  • NL 生成图表/看板
  • DeepSeek + 41 MCP 工具
